# JASCA AI 백그라운드 분석 및 보고서 내보내기 설계

## 목표

JASCA의 모든 AI 기능을 페이지 이동과 무관하게 실행하고, 완료 또는 실패 시 사용자 알림으로 결과를 안내한다. AI 결과는 한국어 보안 보고서 구조로 생성하며 PDF와 DOCX로 정상 다운로드할 수 있어야 한다. Harbor 관리자 화면은 운영자가 이해하기 쉬운 한국어로 통일한다.

## 확인된 문제

- AI 실행은 브라우저의 단일 HTTP 요청이 끝날 때까지 기다리는 구조여서 사용자가 페이지를 이동하면 진행 상태와 결과 확인 흐름이 끊긴다.
- PDF/DOCX 다운로드 실패 메시지가 화면에 표시되지 않고 접근성 전용 영역에만 있어 사용자는 클릭해도 반응이 없다고 느낀다.
- AI 기본 프롬프트, 결과 패널 및 보고서 생성 코드 일부의 한국어 문자열이 깨져 있다.
- 현재 보고서는 AI 응답 원문 중심이며 스캔 개요, 심각도 통계, 발견 항목 표, 조치 우선순위가 일관된 문서 구조로 정리되지 않는다.
- Harbor 관리자 및 사용자 스캔 화면에 영어 문구가 남아 있다.

## 검토한 접근법

### 1. 브라우저 요청을 그대로 유지하고 전역 상태만 보존

구현량은 작지만 새로고침, 로그아웃, 네트워크 단절 및 서버 재시작에 취약하다. 알림에서 과거 결과로 안정적으로 이동하기 어렵다.

### 2. Redis 전용 작업 큐 도입

다중 인스턴스 확장에는 유리하지만 현재 요구사항을 위해 별도 큐 라이브러리, 작업 관리 화면과 장애 복구 체계를 추가해야 한다. 폐쇄망 반입물과 운영 복잡도가 커진다.

### 3. PostgreSQL 기반 AI 작업 상태와 API 내부 작업 실행

기존 `AiExecution`과 PostgreSQL을 활용한다. 작업 상태가 영속화되어 페이지 이동과 새로고침에 안전하고 별도 외부 구성요소가 필요 없다. 현재 Docker 단일 컨테이너와 향후 K8s 모두에서 사용할 수 있도록 작업 선점과 stale 작업 복구를 포함한다. 이 방식을 채택한다.

## AI 작업 흐름

1. 사용자가 AI 기능을 실행하면 API는 `AiExecution`에 `QUEUED` 작업을 먼저 저장하고 작업 ID를 즉시 반환한다.
2. API 작업 실행기가 작업을 원자적으로 `RUNNING` 상태로 선점한 뒤 사내 AI 모델을 호출한다.
3. 사용자는 즉시 다른 페이지로 이동할 수 있다. 웹은 작업 ID를 전역 상태에 저장하고 필요한 화면에서 상태를 조회한다.
4. 성공하면 결과, 모델, 토큰, 실행 시간을 저장하고 `SUCCESS`로 변경한다. 실패 또는 제한 시간 초과도 각각 상태와 원인을 저장한다.
5. 성공 시 `AI 분석이 완료되었습니다`, 실패 시 `AI 분석에 실패했습니다` 사용자 알림을 한 번 생성한다.
6. 알림 링크는 `/dashboard/ai-results/{executionId}`로 이동하며 사용자는 결과, 사용된 프롬프트, PDF/DOCX 다운로드를 확인한다.
7. 서버 재시작 시 `QUEUED` 작업을 다시 처리하고, 일정 시간 이상 `RUNNING`인 stale 작업은 안전하게 재시도하거나 실패 처리한다.

기존 AI 실행 버튼과 패널은 유지하되, 완료까지 현재 페이지를 점유하지 않는다. 현재 페이지에 머무르는 경우 상태 조회 후 결과 패널을 자동으로 열 수 있고, 이미 이동한 경우에는 페이지를 강제로 전환하지 않고 알림만 갱신한다.

## AI 프롬프트와 결과 구조

모든 기본 프롬프트에 다음 공통 규칙을 적용한다.

- 한국어로 작성하고 `<think>`, reasoning, 내부 추론 과정은 출력하지 않는다.
- 제공된 컨텍스트에 없는 CVE, 수치, URL은 추측하지 않는다.
- Markdown 제목과 표를 사용하며 JSON 원문을 그대로 출력하지 않는다.
- 결과가 없는 항목은 `발견되지 않음`, 정보가 없는 항목은 `확인 필요`로 구분한다.
- 스캐너 종류가 Trivy, Checkov, ZAP, Semgrep 중 무엇인지 명시하고 각 결과의 의미를 혼동하지 않는다.

스캔 분석과 보고서 생성은 다음 목차를 사용한다.

1. 분석 개요
2. 핵심 요약
3. 심각도 및 정책 위반 현황
4. 발견된 취약점 또는 정책 위반 요약
5. 라이선스 및 기타 보안 이슈
6. 조치 우선순위
7. 결론과 다음 단계

발견 항목 표는 `ID`, `취약점/정책 이름`, `심각도`, `대상`, `상세 위치`, `참조 정보`를 기본 열로 사용한다. 기본 AI 결과와 기본 보고서는 Critical/High를 우선하여 최대 20건을 표시하고 전체 건수와 생략 건수를 함께 보여준다.

대시보드 요약, 프로젝트 분석, 취약점 조치 가이드, 정책 해석 등 다른 AI 기능은 같은 문체와 검증 규칙을 공유하면서 기능에 필요한 전용 목차를 사용한다.

## PDF/DOCX 보고서

- PDF와 DOCX는 동일한 보고서 모델을 사용해 내용 차이를 방지한다.
- 표지, 생성 정보, 분석 범위, 핵심 요약, 통계, 발견 항목 표, 조치 계획, 근거 순서로 구성한다.
- 한국어 글꼴을 명시적으로 사용하고 표의 행이 페이지 경계에서 읽기 어렵게 잘리지 않도록 처리한다.
- 기본 다운로드는 상위 20건을 포함한다.
- 사용자가 `전체 항목 보고서 생성`을 요청하면 저장된 실행 컨텍스트에서 전체 항목을 포함한 PDF 또는 DOCX를 별도로 생성한다. 전체 보고서는 사용자의 명시적 요청 없이는 생성하지 않는다.
- 다운로드 버튼은 진행 상태를 표시하며 성공 시 실제 파일명을 안내한다. 실패 시 HTTP 상태와 사용자가 취할 수 있는 조치를 화면에 표시한다.
- API는 올바른 `Content-Type`, `Content-Length`, `Content-Disposition`을 반환하고 빈 파일을 성공으로 처리하지 않는다.

## 권한과 보안

- 실행자는 자신의 AI 결과만 조회하고 다운로드할 수 있다.
- 조직 관리자는 같은 조직 사용자의 결과를, 시스템 관리자는 전체 결과를 조회할 수 있다.
- 알림과 결과 페이지에서도 동일한 소유권 검사를 적용한다.
- AI 컨텍스트의 기존 개인정보 마스킹을 유지한다.
- 전체 보고서 생성은 기존 분석 결과와 저장 컨텍스트만 사용하며 외부 AI 호출을 다시 수행하지 않는다.

## Harbor 한국어화

관리자 Harbor 연동 화면의 제목, 설명, 활성화 설정, Robot Account, 토큰, 허용 프로젝트, 기본 JASCA 프로젝트, Webhook 설정, 저장/연결 테스트 버튼, 성공/오류 문구를 모두 한국어로 변경한다. 사용자 스캔 화면의 Harbor 프로젝트·저장소·아티팩트 선택 문구와 오류 메시지도 같은 용어로 통일한다.

고유 제품명과 프로토콜 표기인 `Harbor`, `Robot Account`, `Webhook`, `Authorization: Bearer`, URL 및 Digest는 원문을 유지하고 설명을 한국어로 제공한다.

## 오류 처리

- AI 제공자 연결 실패, 제한 시간 초과, 권한 오류와 보고서 생성 실패를 구분하여 저장하고 표시한다.
- AI 작업 실패 알림에는 실패 원인 요약과 결과 상세 링크를 포함한다.
- 다운로드 실패는 버튼 주변에 보이는 오류 메시지와 재시도 버튼을 제공한다.
- 알림 생성 실패가 AI 결과 저장을 되돌리지는 않으며 서버 로그에 별도로 기록한다.

## 테스트

- API 단위 테스트: 작업 생성, 상태 전이, 소유권, stale 복구, 완료/실패 알림 중복 방지.
- 프롬프트 테스트: 한국어 공통 규칙, 스캔 보고서 목차, 상위 20건 제한, 추론 내용 제외.
- 내보내기 테스트: PDF/DOCX 시그니처, 파일 크기, 응답 헤더, 한글 문자열, 표 데이터, 기본/전체 범위.
- 웹 테스트: 다운로드 성공/실패 표시, 페이지 이동 후 작업 상태 유지, 알림 링크 이동, Harbor 한국어 문구.
- 브라우저 통합 테스트: AI 실행 직후 다른 페이지 이동, 완료 알림 확인, 결과 페이지 이동, PDF/DOCX 실제 다운로드 및 문서 열기.
- 회귀 테스트: 기존 Trivy, Checkov, ZAP, Semgrep 스캔과 알림 기능에 영향이 없는지 확인.

## 완료 기준

- AI 실행 직후 다른 페이지로 이동해도 서버에서 작업이 완료된다.
- 성공/실패 알림이 한 번만 생성되고 결과 상세 페이지로 이동한다.
- PDF와 DOCX가 실제로 다운로드되고 한국어와 표가 정상적으로 표시된다.
- 기본 문서는 상위 20건, 사용자 요청 시 전체 문서를 생성한다.
- AI 화면과 Harbor 연동 화면에 깨진 한글이나 불필요한 영어 안내가 남지 않는다.
