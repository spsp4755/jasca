# ZAP 안전 운영 프로필 설계

## 목적

폐쇄망에서 JASCA의 ZAP 웹 검사를 운영할 때, 사용자가 임의의 URL을 검사하거나
인증 정보가 남는 위험을 줄인다. ZAP 검사는 계속 Passive Spider Scan만 지원한다.

## 범위

### 관리자

- 기존 ZAP 전역 설정에 이름 있는 대상 프로필을 추가한다.
- 프로필은 이름, 설명, 허용 URL 패턴, 차단 URL 패턴, 최대 실행 시간, 알림 기준을 가진다.
- 프로필의 허용 패턴은 한 개 이상이어야 한다.
- ZAP 서버 연결 테스트는 현재 URL/API Key 설정으로 버전 조회만 수행한다.

### 사용자

- ZAP 검사 시작 시 하나의 활성 대상 프로필을 선택한다.
- 입력 URL은 선택한 프로필과 기존 전역 정책을 모두 통과해야 한다.
- 인증 방식은 기존처럼 없음, Cookie, Authorization Header만 지원한다.
- 인증 값은 요청 처리 중에만 사용하고 DB, 결과 JSON, 감사 로그, 브라우저 저장소에 기록하지 않는다.

### 결과와 감사성

- 결과 evidence에는 프로필 이름, Passive 모드, 대상 URL, 시작/종료 시각, 실행 시간,
  인증 방식 이름만 기록한다. 인증 값은 기록하지 않는다.
- 취소·시간 초과·정책 위반은 기존 API 오류로 종료하며 결과를 만들지 않는다.

## 데이터와 흐름

1. 관리자가 `zap` 설정에 대상 프로필을 저장한다.
2. 사용자가 프로필과 URL을 선택해 스캔을 요청한다.
3. API는 전역 allowlist/denylist와 프로필 allowlist/denylist를 모두 검증한다.
4. API는 프로필별 최대 실행 시간을 전역 값보다 작게만 적용한다.
5. ZAP Spider 완료 후 Passive Alert를 가져오고, 인증 헤더 규칙을 즉시 제거한다.

## 안전 경계

- `active` 모드는 API에서 계속 거부한다.
- 프로필이 없거나 비활성화된 프로필은 실행할 수 없다.
- 프로필 URL 범위는 `http` 또는 `https` 및 기존 wildcard 규칙을 사용한다.
- 연결 테스트는 ZAP `core/version` API 외의 스캔 동작을 수행하지 않는다.

## 제외 범위

- ZAP Context/사용자/폼 기반 자동 로그인 설정
- 인증 값을 서버에 저장하거나 재사용하는 기능
- Active Scan, 공격 강도, Spider 깊이, 요청 속도 조절
- 별도 스케줄러 또는 승인 워크플로

## 검증

- 프로필 미선택, 비활성 프로필, 범위 불일치, Active 요청은 ZAP 호출 전에 거부한다.
- 인증 값이 evidence에 포함되지 않는지 확인한다.
- 연결 테스트 성공/실패를 단위 테스트한다.
- API 전체 테스트, API/웹 빌드, Docker AMD64 빌드와 로컬 UI 흐름을 확인한다.
