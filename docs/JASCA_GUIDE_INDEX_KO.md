# JASCA 사용 설명서 인덱스

이 문서는 JASCA를 처음 사용하는 사용자와 운영 관리자가 필요한 문서를 빠르게 찾을 수 있도록 정리한 안내서입니다.

## 문서 구성

| 문서 | 대상 | 내용 |
| --- | --- | --- |
| [JASCA 사용자 가이드](./JASCA_USER_GUIDE_KO.md) | 일반 사용자, 개발자, 보안 담당자, 프로젝트 관리자 | 로그인, 대시보드, 프로젝트, 스캔, 취약점, 라이선스, 정책, Trivy DB, API 토큰, 연동 가이드, 알림, 개인 설정 사용법 |
| [JASCA 관리자 가이드](./JASCA_ADMIN_GUIDE_KO.md) | 시스템 관리자, 조직 관리자, 보안 관리자 | 조직, 사용자, 권한, 프로젝트, 정책, 예외, 워크플로우, 컴플라이언스, AI, Trivy, 알림, CI/CD, 감사 로그, DB/스키마, API Explorer, 운영 배포 관리 |

## 빠른 시작 순서

1. 관리자는 [관리자 가이드](./JASCA_ADMIN_GUIDE_KO.md)를 보고 조직, 사용자, 권한, Trivy DB, AI 설정을 먼저 준비합니다.
2. 사용자는 [사용자 가이드](./JASCA_USER_GUIDE_KO.md)를 보고 프로젝트를 선택하거나 생성한 뒤 스캔을 업로드합니다.
3. 스캔 결과가 쌓이면 취약점, 라이선스, 정책 위반, 리포트를 확인합니다.
4. CI/CD나 외부 시스템 연동이 필요하면 API 토큰을 발급하고 연동 가이드의 curl 예시를 적용합니다.
5. 폐쇄망 운영 환경에서는 Trivy DB와 AI 모델 연결 상태를 주기적으로 점검합니다.

## 주요 역할

| 역할 | 권장 사용자 | 주요 권한 |
| --- | --- | --- |
| SYSTEM_ADMIN | 전체 시스템 운영자 | 전체 조직, 사용자, 설정, DB, 감사, AI, Trivy 관리 |
| ORG_ADMIN | 조직별 관리자 | 소속 조직의 사용자, 프로젝트, 정책, 예외, 토큰 관리 |
| SECURITY_ADMIN | 보안 운영자 | 취약점, 정책, Trivy, 수동 취약점, 예외, 컴플라이언스 관리 |
| PROJECT_ADMIN | 프로젝트 책임자 | 프로젝트 단위 스캔, 취약점 처리, 정책 예외 요청/관리 |
| DEVELOPER | 개발자 | 스캔 업로드, 취약점 확인, 담당 취약점 처리 |
| VIEWER | 조회 사용자 | 대시보드, 프로젝트, 스캔, 취약점, 리포트 조회 |

## 운영 시 반드시 확인할 항목

| 항목 | 확인 위치 | 권장 점검 |
| --- | --- | --- |
| Trivy DB | 관리자 > Trivy 설정, 사용자 > Trivy DB | DB 파일 존재, UpdatedAt, Java DB 존재 여부 |
| 스캔 기본 모드 | 스캔 업로드 화면 | 기본값은 Filesystem(fs), rootfs/image/rpm은 대상에 맞게 명시 선택 |
| 업로드 용량 | 배포 env | `TRIVY_UPLOAD_MAX_BYTES`가 운영 파일 크기에 충분한지 확인 |
| AI 모델 | 관리자 > AI 설정 | 사내 모델 URL, 모델명, API Key, 컨테이너 내부 네트워크 연결 확인 |
| 데이터 보존 | `/app/jasca/pgdata`, `/app/jasca/redis` | 재배포 전 백업 또는 볼륨 보존 여부 확인 |
| 권한 분리 | 관리자 > 사용자/권한/조직 | 조직별 접근 범위와 역할이 맞는지 확인 |
